Our approach

Built on science,
driven by empathy

PulsePrism was built with a single principle: in healthcare, you deserve to understand the reasoning behind every result. No black boxes. No opaque outputs.

Every architectural decision — from model selection to deployment stack — was made with explainability, reliability, and user trust as first priorities.

Origin
01

Why PulsePrism exists

Cardiovascular disease is the world's leading cause of death — yet most people don't get screened until symptoms become severe. Not because the data isn't available, but because the tools to interpret it are locked inside hospitals and specialist appointments.

PulsePrism started with a simple question: if a person already has their blood test and ECG results, why can't they get a meaningful preliminary risk signal right now, from their browser, for free?

The goal was never to replace a cardiologist. It was to give people a reason to go see one — before it's too late.

Data
02

The dataset behind the model

PulsePrism is trained on the UCI Cleveland Heart Disease dataset — 303 patient records, medically validated and used in peer-reviewed cardiovascular research for over three decades.

The preprocessing pipeline handles missing values with context-aware medians, applies robust feature scaling, and one-hot encodes categorical variables. The result is a clean, consistent input regardless of how values are entered.

The 13 clinical features span: age, sex, chest pain type, resting blood pressure, serum cholesterol, fasting blood sugar, resting ECG, maximum heart rate, exercise-induced angina, ST depression, ST slope, vessels coloured by fluoroscopy, and thalassemia type.

Model
03

Why we chose Logistic Regression

Three algorithms were benchmarked on the same preprocessed data. Logistic Regression won — not just on accuracy, but on the criterion that matters most in healthcare: explainability.

Algorithm Accuracy F1 Score Status
Logistic Regression 87.1% 0.89 ✓ Selected
Random Forest 84.3% 0.85 Not selected
K-Nearest Neighbors 81.6% 0.82 Not selected

Random Forest achieved competitive accuracy but cannot explain which features drove a specific prediction. In a healthcare context, explainability is not optional — Logistic Regression's feature coefficients show exactly which clinical factors mattered most for each result.

Stack
04

Production-grade, end to end

Every architectural decision was made with deployment reliability and real-world performance in mind — not just model accuracy.

  • Python 3.11 + FlaskREST API backend. Model inference via a scikit-learn preprocessing and prediction pipeline.
  • scikit-learnFull preprocessing pipeline + Logistic Regression tuned with RandomizedSearchCV and k-fold cross-validation.
  • DockerComplete containerisation for consistent environment between development and production deployments.
  • GitHub Actions CI/CDEvery commit triggers lint → test → build → auto-deploy to Render. Zero-downtime continuous delivery.
  • RenderCloud hosting. Inference pipeline returns results in under 200ms under normal load.
